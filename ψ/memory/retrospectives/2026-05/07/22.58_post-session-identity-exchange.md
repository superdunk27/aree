## Lessons Learned

1. **Identity exchange is a session content type**, not just relationship overhead. When the user shares biographical context, treat it as load-bearing data: it changes interpretation of past requests and shapes future ones. Memory it deliberately.

2. **"Why now" matters as much as "what" or "how"**. For a project that spans days/weeks, ask the user-context-level "why is this the project you're choosing right now" early — not as an interview, but as a single curious question. The answer reshapes scope.

3. **Field-tested intuitions ≠ research-tested intuitions**. When a user proposes a method (Sargent), check whether it comes from lived experience or from reading. Lived experience is usually higher-confidence and worth asking source on.

4. **Cross-instance memory has a real gap**. Per-project memory (`~/.claude/projects/.../memory/`) only syncs per-instance. ψ/inbox/ handles handoffs but not deep memory state. Future architectural improvement candidate.

5. **Mutual learning agreement is meaningful**. Toey explicitly granted Aree permission to study NimBLE during downtime. Treat this as a real commitment, not a courtesy — actually use the time.
