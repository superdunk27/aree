## Lessons Learned

1. **Validator independence > tool prestige**. For accuracy validation, prefer methods that measure via different physical principles (Sargent uses reach difference, mat uses flight time — these can't fail in the same way). Same-principle validators (Chronojump uses flight-time too) only validate protocol implementation, not measurement correctness.

2. **Verify install before claiming success**. winget exit code 0 ≠ binary in PATH. Always run `<tool> --version` or equivalent to confirm. (Workaround for arduino-cli: full path `"/c/Program Files/Arduino CLI/arduino-cli.exe"` since shell PATH wasn't refreshed.)

3. **Arduino sketch convention is strict**: a `.ino` file must live in a folder of the same name. Skipping this was a self-inflicted restructure. Convention check before file creation, not after.

4. **MCP tool misconfig stays silent**: oracle-v2 had been "failed to connect" for unknown duration before this session. Manual verification of `claude mcp list` health states matters.

5. **Cross-instance handoff via `ψ/inbox/` works**: VS Code extension pushed Day 2 work, this Claude Code CLI instance picked up cleanly via `git pull`. Worth noting as a confirmed pattern.
