## Lessons Learned

1. **Always check existing solutions before recommending DIY architecture.** 30 minutes of `gh search` could have saved 1 architecture pivot. The Chronojump ecosystem was discoverable from the very first research query — I just didn't run it.

2. **Reach height ≠ COM displacement.** Vertical jump test (touch wall) gives a different number than physics-formula `h = g·t²/8` (which gives center-of-mass rise). Sub-elite athletes typically jump 50-80cm reach but 35-55cm COM displacement. Without arm swing (single-arm Sargent test), the two converge.

3. **Accuracy fear is upstream of feature decisions.** Toey didn't care about force curves or BLE or any feature until he trusted the measurement. Frame conversations around accuracy first, features second.

4. **License-aware folder structure from day 0.** Separating `firmware/` (forced GPL by adapting Chronojump code) from `pwa/` (clean-room, can be closed) at the folder level prevents derivative-work problems later. Free move, do it always.

5. **Chronopic 4 protocol details for next session**: 115200 baud (NOT 9600), text format `[command]:[argument];`, sensor output `<index>:<phase_duration>\n`, INPUT_PULLDOWN (pressed = HIGH), debounce 10000μs. BLE Service UUIDs in `Communication.md`.
