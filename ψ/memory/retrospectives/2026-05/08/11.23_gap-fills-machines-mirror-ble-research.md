## Lessons Learned

1. **Cross-instance state needs git, not chat memory or per-instance files.** Two practical artifacts established today: `ψ/active/machines.md` (per-machine inventory of skills/MCP/tools, hostname-keyed) and `ψ/memory/personal-context.md` (mirror of durable per-instance memory like user profile, feedback rules, project framings). Pattern: anything Aree-on-a-different-machine would benefit from = belongs in repo, in ψ/. Anything purely about this conversation's transient state = stays in chat. Per-instance memory is *neither* repo nor chat — it's a third tier that needs explicit mirroring discipline.

2. **MCP install requires Claude Code restart before tools become callable.** `claude mcp add` succeeds and `claude mcp list` shows ✓ Connected, but the active Claude Code session loaded its MCP config at startup and won't see new MCPs until restart. Smoke test BEFORE restart can only verify the connection-handshake level (server starts, registers); tool-level smoke test (calling actual MCP tools from the agent) only works AFTER restart. This affects setup-flow expectations and should be communicated upfront.

3. **A good smoke test produces an artifact you'll use.** The Context7 NimBLE smoke test pulled a NimBLE GATT-server skeleton; that skeleton became the basis for the Phase 1.5 BLE design doc 90 minutes later. Compare to a pure ping-style smoke test (e.g., asking Context7 for "react hooks") which would have verified function but produced nothing usable. When testing capabilities, choose a test that doubles as work toward an upcoming task.

4. **Self-prompted tool-choice reflection should be proactive, not user-prompted.** When a primary tool fails or produces partial output, automatically surface the alternatives table (other tools available, their fit for the task, trade-offs) before continuing. Today this would have meant flagging PMC-blocked + listing Playwright/Firecrawl/alternative-URL options for Toey to pick. Instead I worked around silently and he had to externally ask the meta-question.
