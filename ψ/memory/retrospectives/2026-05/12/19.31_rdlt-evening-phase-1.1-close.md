## Honest Feedback

**1. Reactive catch, not reflexive verify** — I caught the manifest drift before outputting wrong information, but only by pausing mid-execution to question my plan. The goal state is "verify on filesystem first, then claim state" as the default reflex. The way to get there is procedural, not aspirational: when reading machines.md sections that name installed software, immediately run a parallel `ls`/`Test-Path` check on the path. Not "if it feels uncertain" — always. Saved as the lesson today.

**2. Edit count was higher than needed** — I made 5 sequential Edit calls on machines.md for changes that could have been bundled into fewer edits (RDLT section had 3 separate changes that share context: last-updated, current-state, gaps, history). Each Edit triggered a hook reminder cycle. If I'd planned the edits as one larger block (e.g., rewrite the RDLT section in one Edit with multiple replacements), it would have been faster and quieter. Not a correctness issue — a flow efficiency issue.

**3. ssh-test verbatim instead of via alias** — I tested `ssh aree` by overriding `RemoteCommand=none`, not by running the actual `ssh aree` command interactively. Reason: non-interactive shell can't `tmux attach`. But this means I haven't actually verified end-to-end that `ssh aree` from Toey's PowerShell drops into the tmux session correctly. Next interactive session at RDLT, Toey should run `ssh aree` once to confirm the full flow. If RemoteCommand syntax is wrong (e.g., the `||` falls through unexpectedly), I haven't caught it.

A smaller fourth: I wrote `IdentityFile ~/.ssh/id_ed25519` in the config and Windows OpenSSH handled `~/` expansion correctly in the BatchMode test. But on some Windows OpenSSH builds `~/` resolution is finicky. If something breaks, that's the first thing to substitute with `$env:USERPROFILE`-style absolute path.

**Primary** (saved to `ψ/memory/learnings/2026-05-12_filesystem-trumps-manifest-claim.md`): Manifest drift is a recurring pattern. Pattern worth keeping for any cross-machine action that needs a different machine's hand.

The persistence layer (retro + lesson files committed in repo) is doing real work — it shifted the immediate-next-session offer menu, which shifted user pick, which shifted my behavior. Worth noticing when retros pay off in concrete sessions, not just in feel.
