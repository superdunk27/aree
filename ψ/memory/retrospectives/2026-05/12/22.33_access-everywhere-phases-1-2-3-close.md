## Key Architecture Decisions

1. **Server-side install script over multi-line PowerShell paste**: `aree-install.ps1` on aree-home + 2-line `scp + run` on the client. PowerShell paste was the biggest friction in this session; this pattern bypasses it. DESKTOP-CE4H6GT will use the same flow.
2. **`tmux new-session -A` over manual attach/new fallback**: atomic, idempotent, no race with `exec` replacement bug.
3. **Source-from-git dotfiles, not symlinks**: HOME files are thin one-liners that source ψ/dotfiles/*.  Pull from git → reload → done. Cleaner blame trail, no broken-symlink edge cases.
4. **Tailscale Serve (not Funnel)**: tailnet-only HTTPS via auto-cert. Toey ruled out public exposure on day 1.
5. **Windows hosts file as DNS fallback**: pragmatic. Tailscale's `--accept-dns` should have worked but didn't on this network (likely Group Policy or static adapter DNS). Logged the gotcha; will likely re-encounter on DESKTOP.
6. **Skipped Phase 4 (Quick captures) and Hardening 5 (browser bookmark)**: Toey explicitly said "ใช้ใน termius เอา" — mobile UX preference is Termius, not browser. Browser channel exists for emergencies / borrowed devices.

## Honest Feedback (3+ friction points)

1. **PowerShell vs CMD vs Windows Terminal vs OpenSSH on Windows is a UX maze**. The banner is subtle (`Microsoft Windows [Version...]` vs `Windows PowerShell`), the prompt is subtle (`C:\Users\toey>` vs `PS C:\Users\toey>`). Toey isn't an OS-shell expert; he picked the first thing the Start menu offered. The right next-time move is **always include a one-line shell sanity check ("paste this first: `Write-Host test`") at the top of any multi-line Windows paste block**. Not "be careful" — proof.

2. **Multi-line PowerShell paste was always going to bite us**. The `>>` continuation prompt + Toey's natural "let me paste it again" instinct = guaranteed broken block. I knew PowerShell behaves this way and didn't preempt it. Lesson: **anything more than 3 lines on Windows → write to file, scp, run from file**. The aree-install.ps1 pattern that closed Phase 1 *should have been the default pattern from line 1*.

3. **Tailscale DNS adoption is fragile on Windows**. `tailscale set --accept-dns=true` is documented as "Use Tailscale's DNS" but it didn't apply on RDLT — ISP IPv6 DNS server stayed authoritative. Could be Group Policy, could be Windows network adapter ordering, could be IPv6 priority. The hosts file fallback is fine for one URL on 2 machines but doesn't scale if we ever add subdomains. **Worth digging into the actual mechanism once instead of papering over it twice** (DESKTOP will hit the same).

4. **I almost wasted a turn on Hardening 5 (browser bookmark)** until Toey told me to stop. The plan said "bookmark URL on every machine" — but Toey's actual mobile workflow uses Termius, not browser. The plan was inferred; the user's actual usage pattern is different. **Lesson: plans assume usage patterns; verify usage patterns at decision points, not just at planning time.**

## Lessons Learned

1. **Multi-line Windows shell paste is fragile — write to file, scp, run from file by default.** PowerShell `>>` continuation + user "re-paste" instinct = broken blocks. The `aree-install.ps1` pattern is now the canonical fleet pattern for any Windows install step over 3 lines. Saved as standalone lesson in `ψ/memory/learnings/2026-05-12_windows-multiline-paste-pattern.md`.

2. **`apt install` packages can ship enabled root services that conflict with user services**. Discovered with ttyd: `ttyd.service` runs as root with `/bin/login` on port 7681 — silently breaking our user-level `aree-web.service`. Always check `systemctl list-units --all | grep <package>` after `apt install` for new system services. Mask anything we're not using.

3. **Tailscale's `--accept-dns` doesn't always win against Windows network adapter DNS**. ISP IPv6 DNS can persist. `nslookup ... 100.100.100.100` confirms Tailscale DNS works; the gap is Windows adoption. Hosts file is the reliable workaround but doesn't scale. Pattern likely repeats on DESKTOP-CE4H6GT.

4. **systemd linger + persistent service config + git-backed dotfiles = unattended power-cut recovery in ~4 minutes**. This is the home-server architecture from 2026-05-08 + the deployment from 2026-05-11 paying off. Worth naming the achievement.
