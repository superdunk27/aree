## Honest Feedback

Three friction points, ranked by how much they cost the session:

**1. Identity-from-context (4th-instance manifest drift)** — I told Toey "this is RDLT" without running
`hostname`. CLAUDE.md prescribes the exact opposite. I knew the rule and didn't follow it because the
*answer felt obvious from recent context*. The lesson the project has been writing all week — manifest
drift between "what's documented" and "what's actually there" — landed in *my own behavior* and I still
needed Toey's prompting to catch it. The remedy is procedural, not cognitive: hostname/skills/MCP check
goes at the front of any machine-state claim, before the answer. Not after.

**2. Tool silence during long operations** — winget install, ~1 min, zero updates. Toey called it out
explicitly. This isn't subtle; it's a known communication pattern (the prompt itself says "give short
updates at key moments" and "silent is not"). I should pre-announce expected duration *before* starting,
and post a "still running" if the elapsed time exceeds ~30s. The `description` field on Bash is for
operators reviewing the tool log; it never reaches the user. Only text between tool calls does.

**3. Stale-state recap** — opened the session with `/recap` and answered confidently about "last session"
when origin was 5 commits ahead. CLAUDE.md cross-instance rule literally says "git is the only reliable
sync layer" — and I didn't `git fetch` before reading state. The downstream cost was Toey had to do a pull,
and I had to redo the recap with corrected context. The cheap fix: any recap that reads "what's the world
look like" should `git fetch --all` (or even `git pull` after confirming clean) as Step 0. Currently the
script doesn't, and I didn't compensate.

A fourth, smaller one: I tried `ssh-keyscan` to prefetch the host key but didn't know in advance that
Windows OpenSSH's keyscan doesn't speak the newer post-quantum KEX that Ubuntu 26.04's sshd defaults to.
Not fatal — first-time `ssh aree` will still trust prompt — but a known-unknown that bit me.

## Lessons Learned

**Primary**: Verify-before-claim, especially for machine identity. The cure CLAUDE.md prescribes
(`$env:COMPUTERNAME` at session start, hostname check before any cross-machine claim) is not optional. It's
the structural answer to the recurring "documented ≠ actual" drift the project has been logging since
2026-05-08. This time the drift was *in me*, not in a manifest file. See learning file dated 2026-05-12.

**Secondary (process)**: When a tool will take >30s, pre-announce the expected duration *before* the call,
not after. And post an interim update if it runs longer than expected. Saved as a feedback memory in
`personal-context.md` mirror.

**Tertiary**: For `tailscale ssh` to work identity-based (no OpenSSH keys), the server side must opt in
with `tailscale up --ssh`. aree-home didn't, so `tailscale ssh` falls back to wrapping the OS ssh client.
If the goal is to bypass OpenSSH keys entirely, enable Tailscale SSH server-side first.
