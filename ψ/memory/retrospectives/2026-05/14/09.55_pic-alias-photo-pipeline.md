4. With ssh config working, `scp file aree:inbox/` is short, but Toey would still need to remember the path to the camera roll AND the latest filename each time.

Final design: a 3-line script `~/p` (`cd ~/cam`, `f=$(ls -t|head -1)`, `scp "$f" aree:inbox/`) plus a symlink `~/cam → ~/storage/dcim/Camera` plus an alias `pic=~/p`. Eight short setup commands total, each under 35 chars and individually paste-safe. After setup, Toey types `pic`, Enter — newest camera roll photo lands in `/home/toey/inbox/` on aree-home in <1 second, Aree reads it via Read tool. Tested end-to-end with a 1.9 MB JPEG of a Bang Na parking E-Stamp ticket Toey had photographed yesterday. Both directions work.

## Honest Feedback

**Friction 1**: Fifth paste-wrap failure of the same shape in 24 hours. I knew this would break before I wrote it. The mitigation is *not* "remember to use short commands" — it's "make the runtime so short that no doc-recall is needed at all", which is what `pic` ultimately did. The retros and learnings have already named the pattern multiple times; the cure has to live in the artifacts I hand Toey, not in advice to myself.

**Friction 2**: Even after splitting into five `echo >> ~/.ssh/config` commands, one of those wrapped (the `IdentityFile` line, ~50 chars). I'd estimated chat wrap at ~40 chars; the actual cliff is somewhere between 40 and 50 and varies by chat context. Empirical answer: target **≤30 chars per pastable line** for safety margin. The wrap budget isn't a constant; pad accordingly.

**Friction 3**: I let myself slip into "let me explain in detail" register at the start of the soldering walkthrough — laid out four numbered steps, a temperature primer, and a solder-type primer in one big response. Toey hadn't started any of it yet. The response *predated* his action — I was front-loading instructions instead of running a one-step-at-a-time loop. The retro pattern from Termux setup ("paste this short line → confirm → next") works for soldering too and probably should be the default for any multi-step procedure with a beginner user.

## Lessons Learned

Writing the lesson file in parallel: `ψ/memory/learnings/2026-05-14_runtime-not-docs.md` — when a user says "I can't remember how to X", the answer is to compress X into a runtime command short enough that no remembering is needed. Documentation isn't the cure for forgetting; ergonomics is. This generalizes beyond `pic`: any task that the user is going to want to invoke repeatedly should be reducible to a 3-5 character word in their shell, not a recipe to look up.

Empirical note for any future mobile-paste work: **≤30 chars per line**, or use multi-line `echo >> file` building rather than one long command. The ~40-char ceiling I'd assumed is too generous; I'd seen `IdentityFile ~/.ssh/id_ed25519_aree` at ~50 chars wrap silently today.
