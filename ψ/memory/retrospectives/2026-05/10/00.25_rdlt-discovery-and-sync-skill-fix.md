## AI Diary

The whole point of yesterday's manifest-drift lesson is that *documented values aren't facts until verified*. Running `$env:COMPUTERNAME` was a 50ms operation. What surprised me: the `/sync` skill caught its own creator's mistake on the literal first run. ## Honest Feedback

**Friction 1: Recap reliability**. `/recap` rich gave a confident-looking summary that included one false fact (TOEY hostname). I should add `$env:COMPUTERNAME` to the recap output itself — same way it shows git status. One-line addition, eliminates an entire class of "I assumed the manifest" errors. Right now `/recap` reads from manifest; it should also probe ground truth on a few cheap signals.

**Friction 2: Mid-session sync skill discovery was lucky**. The skill format bug (flat vs directory) only surfaced because Toey noticed `/sync` wasn't in the skill list. There's no automatic check that locally-defined skills actually loaded. A `/skills-list --local` mode that compares `.claude/skills/*/SKILL.md` (filesystem) vs the in-context available-skills set (runtime) would catch this kind of drift before it costs a session-restart.

**Friction 3: Three serial PowerShell calls early in the /sync run that should have been parallel**. I split `node -v`, `bun -v`, `npx -v`, `winget -v` into one parallel batch — fine. But `Get-Command arduino-cli` + `Test-Path Chronojump` were chained into one call when they could have run alongside the version checks. Minor latency, but a pattern: I parallelize within a tool call but serialize across them when I shouldn't. The hooks reminded me twice today; the second reminder should have changed my next call's structure, not just been acknowledged.

## Lessons Learned

1. **Skill format = directory + `SKILL.md`** (not flat `<name>.md`). Hot-reload works mid-session — no restart required after `git mv`. (See `ψ/memory/learnings/2026-05-10_skill-format-and-hot-reload.md`.)
2. **Manifest drift confirmed as a pattern** (3 instances in 7 days). All three: documented value treated as authoritative. The cost ranges from "wrong gap analysis" (skill count) to "wrong machine identity" (hostname). The cure is a verify-step, made cheap and automatic. The `/sync` skill is the cure for hostnames; analogous probes belong in `/recap`.
3. **The skill that enforces verification can catch the AI that built it**. Worth noting because next time I'm tempted to think a freshly-built tool "needs more time before its first real catch" — no, it caught a real bug on the first run.
