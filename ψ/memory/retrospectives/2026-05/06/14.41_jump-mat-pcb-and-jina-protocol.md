## Lessons Learned

1. **`WebFetch` returns AI summaries, not raw text.** For technical step-by-step content, this is unreliable. Use `curl https://r.jina.ai/<url>` instead — Jina Reader returns verbatim markdown for free. Confirmed working.

2. **Different summaries from the same `WebFetch` call = unreliable.** If two prompts to the same URL return contradicting structures, do not pick the most plausible — switch tools. Three contradicting summaries for the Chronojump page should have triggered an immediate methodology switch, not a fourth prompt.

3. **Shopee, sites with CAPTCHA, JS-rendered SPAs are blocked at the Jina layer too.** Fallback: ask Toey to paste visible info or screenshot. This came up immediately on the first PCB sourcing attempt.

4. **PCB switch-mat physics**: contact closure depends on (a) PCB stiffness, (b) panel size, (c) tape spacer height, (d) applied force. Rigidity can be compensated by using thinner spacer; do not optimize one variable in isolation. 1.5mm FR4 + 1mm spacer is a workable combination at 30×30cm with bodyweight load.

5. **Chronojump in their own words recommends PCB over rods** — this was visible in the actual page text but never surfaced through `WebFetch` summaries. The recommendation has been there from session 1; I just was not seeing it.
