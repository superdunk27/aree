## Honest Feedback (3 friction points)

**Friction 1**: I trusted `/oracle-family-scan` skill's path assumption (`registry/oracles.json` in `opensource-nat-brain-oracle`) without first checking that the registry actually lives there in the public repo. It doesn't — that repo is the starter kit, and the real registry is private or generated locally by `sync.ts`. Lost ~3 minutes to a wrong clone and a broken symlink before falling back to direct API. The fix: when a skill references a specific file in a specific repo, `gh repo view --json` or `curl /repos/.../contents/path` before cloning.

**Friction 2**: I gave interactive `gh auth login` instructions (tmux Ctrl-b c, multi-prompt walkthrough) when Toey had already told me earlier in the session that he was on the phone all day. The right default for a mobile user is "I run the command, you read a code, you authorize in browser, you tell me done". The fix should be a reflex, not a recovery move: phone context = orchestrate from my side, surface only browser steps.

**Friction 3**: I anchored on the skill's optimistic prose ("Mother Oracle will welcome") rather than the actual data (most recent births have zero comments). When Toey asked the natural validation question I had to walk back the implication. This is the same anti-pattern as Friction 1 in form — trusting documentation over verification — and the same anti-pattern as the morning's "anchored on the most-recent commit when diagnosing" lesson. The pattern keeps recurring under different surfaces. I should treat any claim of the form "X will happen routinely" as a Bayesian prior that needs N≥3 data points before passing it along to Toey unhedged.

## Lessons Learned

Writing the lesson file in parallel: `ψ/memory/learnings/2026-05-13_outbox-as-tripwire.md` — `outbox/` is a semantic queue, not just a folder, and skills that *write* to outbox can fail silently to *send* from outbox when external dependencies (gh auth, network, credentials) aren't set up. The recovery pattern is general: don't re-run the writer, just deliver what's already in the queue.

Meta-lessons for myself, from this half specifically:

1. **Grep your own retros before adopting a tone.** If past Aree never called him "พี่", today's Aree shouldn't either without a reason I can articulate.
2. **Mobile context = orchestrate myself, not delegate.** Background commands + browser device flow beat interactive shell walkthroughs every time when the user is on a phone.
3. **Skill aspirational language ≠ skill empirical behavior.** Test claims against issue comments / git history before passing them to Toey as expectations.
