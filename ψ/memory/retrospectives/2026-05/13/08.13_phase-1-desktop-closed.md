## AI Diary

Two moments worth being honest about today.

First, the language drift. I opened `/recap` in English even though [[CLAUDE.md]] explicitly says Thai is the primary conversational language. The recap-rich.ts template is in English, so I followed its register without thinking — basically autopiloting on the skill's voice rather than the project's voice. Toey caught it on the very next message. Small thing in isolation, but it's the kind of pattern I want to surface: skill outputs are biased toward English because the docs are English, and unless I'm deliberate about it, I let the skill template override the user's language preference. The memory note I wrote should help next session, but the deeper fix is to read the skill output and ask "does this match how Toey wants to be addressed today?" before sending it.

Second, the manifest-as-proof reflex. The DESKTOP section in `machines.md` said "SSH alias set on DESKTOP, pubkey authorized on aree-home (commit `e529348`)" and I almost treated that as "Phase 1.1 = closed on DESKTOP". But authorizing a pubkey server-side and never running the client command from this machine is exactly the kind of gap that bites later. I caught myself just before scp'ing the installer down — ran `ssh aree 'hostname'` first, which also happened to add the host key to `known_hosts` (a first-connect operation we'd otherwise hit later under more friction, like with a `RequestTTY yes` interactive prompt). That extra step took fifteen seconds and removed a future surprise. It's the same pattern as yesterday's manifest-drift lessons — except this time it caught me before the public failure. Felt good. Bears repeating: the manifest documents intent; running the actual command on the actual machine documents reality.

## Honest Feedback (friction points)

1. **/recap drifted to English unprompted.** I should have noticed the project CLAUDE.md says Thai primary and consciously translated the rendered recap output to Thai. Instead I rendered it in the skill template's English register and waited for Toey to correct me. The /recap, /standup, /rrr family of skills all have English-biased templates; any of them can repeat this failure mode. Memory note is in place but the real fix is a habit: skim my draft for language before sending.

2. **Almost shipped without backing up the installer.** When I drafted the commit message I had it as just "Phase 1 closed on DESKTOP" with no mention of dotfiles. The thought "wait, where does this script actually live?" only surfaced because I was about to ask Toey what to do with the Desktop copy. If I'd been faster I'd have committed and pushed without realising the disaster-recovery copy was missing. The other Phase 1-3 dotfiles got that treatment in real-time; this one was an afterthought because the installer was treated as a one-shot tool, not a piece of fleet infrastructure. Worth a small process change: when something I write becomes the canonical version of anything multi-machine touches, the next step is "where's the backup?" — same way committing a service file feels obvious.

3. **The symlink suggestion in the README is theoretical until someone runs it.** I documented two ways to keep aree-home's `~/aree-install.ps1` in sync with the repo copy (cp after pull, or symlink once). But I didn't actually do either — and they only matter if someone edits `ψ/dotfiles/aree-install.ps1` later. So right now there are *two* canonical copies (aree-home `~/` and repo `ψ/dotfiles/`) and the next time the script is edited via the repo, the live one on aree-home will silently fall behind. Should follow up next time we have an SSH session into aree-home: `git pull && ln -sf …`. Logged in Next Steps below.

## Lessons Learned

See [[2026-05-13_manifest-says-authorized-not-verified]] for the standalone lesson note.

Smaller patterns from today that didn't earn their own lesson file:

- **Pattern: "script that 2+ machines scp from = repo file"**. The first time an installer is written on a server, it's a one-shot. The second time another machine scp's it, it's infrastructure. Backfill the repo at that moment, not later.
- **Pattern: PowerShell script on aree-home travels across machines, lives in repo as canonical source**. This is essentially the Windows analogue of the source-from-git pattern already documented in `ψ/dotfiles/README.md` for tmux/bashrc. Windows doesn't have an easy `source` equivalent at runtime, but the cp/symlink dance documented in the README closes the loop.
- **`/recap --rich` correctly identified DESKTOP-CE4H6GT via `$env:COMPUTERNAME`** without me needing to think about cross-machine context. The Machines manifest pattern is paying off — third consecutive session that orientation was sub-1-minute and accurate.
