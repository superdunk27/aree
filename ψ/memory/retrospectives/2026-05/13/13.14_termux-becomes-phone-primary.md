## Honest Feedback

**Friction 1**: I gave Toey a heredoc to set up `~/.ssh/config` even though I knew he'd be copy-pasting from chat into a mobile terminal, which is the worst environment for heredoc fragility. The EOF terminator collecting leading whitespace from markdown indentation was *predictable*. I should have defaulted to `printf` or `nano` from the start. The fact that the second attempt (printf) *also* broke, this time from hard-newlines at markdown soft-wrap points, is a clear pattern: chat-paste-to-phone-shell is hostile to multi-line bash. The mitigation should become reflex — short single-line commands, `nano` for anything multi-line.

**Friction 2**: When Toey said "ก็ยังเป็นอยู่" after the Ctrl-C, I wasn't sure whether "still happening" referred to the heredoc problem, the Thai duplication, or both. I guessed (correctly, it turned out) but I should have asked. Ambiguity in a debugging session compounds — guessing wrong adds 5 minutes.

**Friction 3**: I let two separate things be "in flight" at once near the end — Toey doing the alias setup in a fresh Termux session, while I updated the plan + learning. I told him to switch sessions and run a command, then immediately started editing files. If his command had failed or needed clarification, my attention was already split. For mobile workflows where the user can't quickly re-orient, sequential is safer than parallel.

## Lessons Learned

The technical learning is fully captured in `ψ/memory/learnings/2026-05-13_termius-thai-combining-bug.md` — both the original write-up (Termius render bug + ttyd workaround) and the follow-up (Termux as the right primary, plus the two paste-fragility gotchas). I'm not going to duplicate it here as a separate lesson file.

Meta-lessons for *my own* future sessions:

1. **Ask "where" before "what"** on any bug report. The cheapest clarifying question is which device / which surface / which user — and I keep skipping it in favor of jumping into config inspection.
2. **Don't hand a phone user a heredoc.** Default to `printf` (single line) or `nano` (interactive). Chat-to-mobile-shell paste is hostile to anything that depends on whitespace or line boundaries surviving.
3. **When in doubt, pick the path with the most ergonomics built in.** Termux beat Termius and ttyd-in-Chrome not because it's "better" in some abstract sense, but because it has Ctrl/Esc/Tab/arrows in its toolbar by default. The fewer external dependencies (Hacker's Keyboard, custom configs) a path requires to be daily-usable, the more likely it survives as the primary.
