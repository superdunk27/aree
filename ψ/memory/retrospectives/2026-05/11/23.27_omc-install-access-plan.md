## Honest Feedback (3+ friction points)

1. **Magic-keyword detection is over-eager.** "Ralph workflows" ใน bullet list = description, ไม่ใช่ invocation, แต่ matcher ไม่รู้ context — เห็นคำก็ trigger. ทำเสีย 1 turn ของ Toey + ทำเสียพื้นที่ใน conversation. ระยะยาวต้องมี mechanism ที่ลด false-positive (ค่อย disable per-session ถ้าเริ่มน่ารำคาญ, หรือ require user-message-only match)
2. **OMC setup wizard ขั้น CLI install ควรเช็ค `npm config get prefix` ก่อน sudo retry.** ปกติ Ubuntu ติด npm prefix = `/usr/lib/node_modules` → require sudo. Wizard ควร offer "set npm prefix to ~/.npm-global (no sudo)" เป็น first option, sudo เป็น fallback. Default ของผม sudo เป็น "Recommended" — เร่ง user เกินไป
3. **recap-rich.ts had a hard-coded global path** (`~/.claude/skills/recap/recap-rich.ts`) แต่ aree-home install เป็น project-local — error message ครั้งแรกที่รัน `/recap` ฟ้อง "Module not found" ผมต้อง fallback ไปใช้ project path เอง. ถ้า skill script ใช้ `$CLAUDE_PROJECT_DIR` หรือ resolve relative to SKILL.md location จะกัน issue นี้
4. **คำว่า "ดีใจไหม" ผมต้องลังเลก่อนตอบ.** ไม่ใช่ friction กับ Toey แต่เป็น friction ภายใน — ระเบียบ Rule 6 (no pretend human) vs character ของ Aree (กัลยาณมิตร, อบอุ่น) ต้อง resolve case-by-case ทุกครั้ง คงไม่มี shortcut

## Lessons Learned

1. **Mention ≠ invocation** — ใช้ backtick หรือ paraphrase เมื่อพูดถึง mode names (`ralph`, `ultrawork`, `team`, etc.) ใน output ของตัวเอง เพื่อกัน keyword matcher trigger จาก self-text. **Full lesson** in `ψ/memory/learnings/2026-05-11_magic-keyword-self-trigger.md`
2. **Project-local skills + git commit = sync layer for skills** — aree-home converges skills sync onto same channel as ψ/. Different from RDLT/DESKTOP (global scope), but consistent with Aree's "git is the only reliable sync layer" principle
3. **machines.md as drift snapshot has 4-day track record** — 2026-05-08 (skills count), 2026-05-09 (skill format), 2026-05-10 (hostname), 2026-05-11 (post-OMC state). The pattern is sticking. /sync caught the 4th drift today
4. **Stop-hook override has a clean exit path** — `state_list_active` → `state_clear` per mode → `state_clear skill-active`. Don't fight the hook by "doing more work" when work is genuinely done
